Final Year Project: Function Programming Languages in Ray Tracing in Video Game Engines

This was a project to investigate the use of functional programming languages in the context of ray tracing in video game engines. The main aim was to compare the performance and effectiveness of a Haskell and C++ to implementation of a ray tracer. The C++ implementation was based on the Ray Tracing using the Physically Based Rendering book by Pharr and Humphreys (2004) as a guide. I developed the Haskell version of the ray tracing using a similar structure to the C++ version, then both versions were tested using the same 3D models messuring both time to render and the quality of the final image. The results showed that the Haskell version was slightly slower than the C++ version but the quality of the final image was the same. But the the naiveity of the Haskell version and the potential for improvement that with a further development of the Haskell version it could be faster as the C++ version. This is valdiated by the current standard of the gaming industry where a different functional programming language is used for ray tracing.
